# Project Charter

## Project Name
Perceived Topography Framework

## Purpose
Build a theoretical framework explaining how perceived environmental structures shape behavior — then publish a paper about it.

## Scope
- Develop the core thesis and supporting architecture
- Log discoveries, aha moments, and emerging concepts systematically
- Preserve contradictions as fault lines rather than resolving them prematurely
- Produce a publishable paper as one output of the theory

## Principles
1. **Theory-building first, paper second** — The repository is optimized for theory construction. The paper is a downstream artifact.
2. **Classification before insertion** — Every insight gets classified (Discovery, Aha Moment, Fault Line, Emerging Concept, Definition Change, Counterargument) before being added.
3. **Contradictions are preserved** — Conflicts between discoveries and the concept architecture are never silently resolved. They become fault lines.
4. **The concept architecture is sovereign** — `01_CONCEPT_ARCHITECTURE/` is the most important folder. Everything else exists to support or challenge it.
