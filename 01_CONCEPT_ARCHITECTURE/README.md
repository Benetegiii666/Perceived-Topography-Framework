# 01_CONCEPT_ARCHITECTURE

**Priority: HIGHEST — This is the most important folder in the repository.**

Everything else exists to support or challenge what lives here.

## Purpose
Contains the core theoretical framework: the thesis, supporting theses, definitions, hierarchy, relationships, and open questions that define the Perceived Topography Framework.

## Conflict Protocol
If a discovery or insight conflicts with the architecture defined here:
- Do NOT change these files.
- Do NOT suppress the discovery.
- Open a fault line in `04_FAULT_LINES/`.

## Contents
- `CURRENT_STATE.md` — The "living constitution" — canonical snapshot of what we currently believe
- `CORE_THESIS.md` — The central argument of the framework
- `SUPPORTING_THESES.md` — Secondary claims that extend the core thesis
- `DEFINITIONS.md` — Precise definitions of all framework terms
- `HIERARCHY.md` — How concepts relate in order of priority/dependency
- `RELATIONSHIPS.md` — Maps between concepts (perception ↔ topography ↔ behavior)
- `OPEN_QUESTIONS.md` — Unresolved questions within the architecture itself
