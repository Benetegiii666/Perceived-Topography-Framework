# Hierarchy

How concepts relate in order of priority and dependency.

## Current Hierarchy (v0.1)

```
Reality → Topography → Perception → Optimization → Interaction → Emergent Behavior → Outcomes → Reality
```

Note the cycle: Outcomes feed back into Reality, which reshapes Topography. This is not a linear chain — it is a loop.

## Candidate Hierarchy (from FL_007)

If Information Surfaces and Perception Construction survive scrutiny:

```
Reality → Topography → Information Surfaces → Perception Construction → Optimization → Interaction → Emergent Behavior → Outcomes → Reality
```

This adds a layer between Topography and Perception, making explicit the mechanism by which topography becomes available to the agent.

## Key Dependency Rules

1. **Downstream depends on upstream** — You cannot understand Behavior without understanding Optimization, which requires Perception, which requires Topography.
2. **Governance is not in the chain** — Governance acts *on* the chain (reshaping Topography, designing Information Surfaces) but is not a step within it.
3. **The loop matters** — Outcomes change Reality, which changes Topography, which changes Perception. Behavior reshapes the landscape that shaped it.
