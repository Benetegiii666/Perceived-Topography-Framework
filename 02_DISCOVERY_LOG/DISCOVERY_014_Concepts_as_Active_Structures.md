# DISCOVERY_014: Concepts May Function as Active Organizational Structures Rather Than Passive Categories

**Status:** Candidate — requires operational validation
**Date:** 2026-06-10

## Observation

During operational architecture exploration, concepts evolved from classification artifacts into organizing structures that:

- Guide retrieval
- Group assets
- Support decisions
- Inform agent behavior

The emerging architecture suggests:

```
Artifacts
    ↓
Metadata
    ↓
Concepts
    ↓
Decision Maps
    ↓
Agent Actions
```

where concepts serve as the **primary navigation layer** rather than merely a taxonomy layer.

## Implication

Concept-centric architectures may provide the bridge between information ecosystems and decision-support systems. If concepts are active structures (not passive labels), they function as a form of **information surface design** — organizing reality into perceivable gradients for the optimizer.

This connects to the framework at multiple levels:
1. Concepts shape which gradients become visible (information surface function)
2. Concepts influence how gradients are interpreted (context function)
3. Concepts determine retrieval paths (attention function)
4. A poorly designed concept layer produces different behavior than a well-designed one — even with the same underlying data

## Pressure Test

Would concept-centric retrieval outperform artifact-centric retrieval for real marketing workflows?

- **Artifact-centric:** "Find me the Q3 campaign brief" → returns a document
- **Concept-centric:** "What do we know about customer retention for enterprise accounts?" → returns a synthesized view across multiple artifacts organized by concept

If concept-centric retrieval produces better decisions (D011 test), concepts earn their place as active structures rather than passive taxonomy.
